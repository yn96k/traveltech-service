# Кейс по системной аналитике - сервис совместного планирования путешествий

## Требования

Сервис позволяет пользователю спланировать поездку: подобрать билеты, отели, трансферы, активности. Все бронирования синхронизируются в едином интерфейсе, система выдаёт рекомендации на основе интересов и истории.

### Функциональные требования
- Поиск и бронирование билетов/отелей
- Планирование маршрута путешествия
- Синхронизация расписаний и уведомлений
- Персонализированные рекомендации
- Уведомления о задержках и изменениях
- Обратная связь по поездке
### Особенности проектирования:
- Многопользовательский режим (семейные или групповые поездки)
- Интеграции с внешними API (авиаперевозчики, букинги)
- Обработка платежей, паспортных данных
- Работа с тайм-зонами и международными форматами данных
- Вызовы
- Конфликты расписаний и пересадок
- Персонализация без утечки данных
- Надёжность при отказах внешних API
